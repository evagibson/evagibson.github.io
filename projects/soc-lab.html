Overview

What this project is about

Project Goal

The goal of this lab was to create a controlled technical environment where I could work across infrastructure, networking, system setup, security monitoring, and vulnerability assessment. Rather than focusing on one tool in isolation, the project connected several systems into one practical workflow.

This made the lab useful both for technical learning and for showing how core defensive technologies work together inside a security-oriented environment.

What I Built

  • A pfSense firewall setup with rules and network segmentation
  • Ubuntu systems for services, tooling, and administration
  • Windows Server components inside the lab environment
  • Splunk for log collection and monitoring workflows
  • OpenVAS for vulnerability scanning and analysis
  • Docker and Portainer for service deployment and management

Architecture

How the lab was structured

Network Layer

pfSense was used to manage network boundaries, firewall rules, and segmentation. VLAN-based separation supported clearer traffic control and helped create a more realistic defensive lab structure.

Monitoring Layer

Splunk served as the monitoring and analysis layer for lab visibility, enabling log ingestion, review, and security-focused observation.

Scanning Layer

OpenVAS added vulnerability assessment capability, allowing hosts and systems in the lab to be evaluated for exposure and weaknesses.

Systems Layer

Ubuntu and Windows Server were used as core systems inside the lab, with Docker and Portainer supporting deployment and service control.
